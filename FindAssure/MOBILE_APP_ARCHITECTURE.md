### Key Concepts:

1. **Components**: Building blocks of your UI (like LEGO pieces)
2. **JSX/TSX**: A syntax that looks like HTML but creates mobile UI elements
3. **State**: Data that changes over time (like user input, loading status)
4. **Props**: Data passed from parent to child components
5. **Navigation**: Moving between different screens in the app

## ğŸ“ Understanding TypeScript in React Native

TypeScript adds type safety to JavaScript:

```typescript
// Without TypeScript (JavaScript):
const user = { name: 'John', age: 25 };
user.age = '25'; // Oops! Should be number, but no error

// With TypeScript:
interface User {
  name: string;
  age: number;
}

const user: User = { name: 'John', age: 25 };
user.age = '25'; // âŒ Error: Type 'string' is not assignable to type 'number'
```

**Benefits:**
- Catch errors at compile time (before running app)
- Better IDE autocomplete and suggestions
- Self-documenting code (types show what data looks like)
- Easier refactoring (TypeScript finds all places to update)
