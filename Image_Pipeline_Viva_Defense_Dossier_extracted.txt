11. Viva Questions With Model Answers
 
 
Q1. Explain your component in your own words.
 
 
This component is the internal vision backend of FindAssure. It analyzes uploaded item images, extracts 
structured evidence, verifies multi
-
view consistency, supports image retrieval, and reports the results back 
through the Node backend.
 
 
Q2. Why is the app not calling the Python service directly?
 
 
Because Node owns workflow, product rules, and safe fallback behavior. The Python service is specialized for 
vision logic only.
 
 
Q3. What happens when `analyze_pp1` runs?
 
 
It validates that exactly one file was uploaded, saves it temporarily, converts HEIC if needed, calls the shared PP1 
pipeline, stores valid results, returns the analysis, and removes the temp file.
 
 
Q4. Why use YOLO before Florence?
 
 
Because detection should happen first. A good crop makes Florence more reliable and reduces scene noise.
 
 
Q5. Why use Florence after detection?
 
 
Because Florence can extract richer evidence such as OCR, descriptions, and grounded details from the object 
crop.
 
 
Q6. What does the PP1 service orchestrator do?
 
 
It coordinates the full single
-
image flow from detection to evidence extraction, reasoning, embedding 
generation, and fallback handling.
 
 
Q7. Why not fail completely if Gemini is unavailable?
 
 
Because reasoning is optional support logic. The pipeline should still return a usable degraded result when 
possible.
 
 
Q8. What makes PP2 different from PP1?
 
 
PP2 does cross
-
view verification and fusion. It does not just repeat PP1.
 
 
Q9. Why do you need category
-
specific thresholds in PP2?
 
 
Because different item categories behave differently across views, so a single threshold would be too crude.

Q10. What does `get_thresholds` really control?
 
 
It selects the similarity thresholds used to decide whether the uploaded views are likely from the same physical 
item.
 
 
Q11. Why does the search route rerank after FAISS?
 
 
Because nearest
-
neighbor similarity alone is not enough. The system also checks category, OCR, brand, and 
color consistency.
 
 
Q12. Why validate `top_k` and `min_score`?
 
 
Because they affect both safety and usefulness. Extreme values can make search noisy or wasteful.
 
 
Q13. Why is vector indexing in Node treated as non
